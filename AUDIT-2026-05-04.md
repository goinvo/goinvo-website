# GoInvo Website + Sanity Studio Audit — 2026-05-04

Walkthrough of the live dev build (`npm run dev`, http://localhost:3000) and `/studio` at desktop (1440px) and mobile (390px viewport). Grouped by severity / area; line refs in the codebase where I could pin them down.

## Bugs (functional)

**Console error on every page that loads the contact form.** React 19 rejects the JSX prop `allowTransparency` on a native `<iframe>`:

> React does not recognize the `allowTransparency` prop on a DOM element. If you intentionally want it to appear in the DOM as a custom attribute, spell it as lowercase `allowtransparency` instead.

Source: `src/components/forms/ContactFormEmbed.tsx:32` passes `allowTransparency` straight through `LoadingIframe.tsx:34` to the `<iframe>`. Reproduces on `/`, `/work`, `/services`, `/contact`, `/about/careers`, and any other surface that mounts the embed. Fix: rename to lowercase `allowtransparency` (the actual HTML attribute) or just drop it — modern iframes are transparent by default when the inner doc has no background.

**404 page renders the Header and Footer twice.** Visit any unknown route under a layout group, e.g. `http://localhost:3000/work/ipsos`. The DOM contains 2 `<header>` and 2 `<footer>` elements; visually you see one header (they overlap because both are sticky) and two stacked footers. Cause: `app/not-found.tsx` lives at the app root and manually renders `<Header />` and `<Footer />`, but Next.js still wraps not-found content in the matched route's layout — the `(main)/layout.tsx` already renders Header (and Footer via `TransitionLayout`). Fix: drop the Header and Footer imports from `app/not-found.tsx` (the layout handles them), or move not-found.tsx inside the `(main)` group.

**Mobile homepage has ~14px of horizontal overflow.** At 390px viewport, `document.documentElement.scrollWidth` = 404. Culprit is the hero "Backlog: Zero" / 3M case study card: the inner panel is `w-[85vw]` inside a `bg-cover` section, and its right edge resolves to 404 — past the viewport — producing a horizontal scrollbar.

**Mobile menu truncates "Open Design"** to "Open Desig" — last letter clipped at the panel's right edge. Either widen the panel or shorten the label.

## Page transition perceived as broken

Every navigation triggers Framer Motion's TransitionLayout fade-in. For ~1–3 seconds after `router.push`, the body content sits at low opacity. On `/work`, `/vision`, `/services`, and case study pages this looks like the page is broken — copy is faint, hero images and card thumbnails are mid-fade. Several screenshots I took caught the page in this state and would have been indistinguishable from broken styling on a slow connection. Recommend either starting from a higher initial opacity, shortening the in-animation, or animating only on exit.

A related symptom: above-the-fold case study card thumbnails on `/work` render as solid grey boxes during the fade, then "pop in" once the user scrolls. Combined with the next two findings this is the worst above-the-fold experience.

## Image performance

Console floods with three classes of warning, all on Sanity-hosted images:

- **`<Image fill />` is missing `sizes`** (~20 instances across `/work`, `/vision`, related). Next.js can't pick the right responsive variant, so it ships unnecessarily heavy assets.
- **LCP image lacks `loading="eager"`** — the work-page hero is detected as the LCP candidate; flagging it eager would improve LCP on first paint.
- **Aspect ratio drift** — several Sanity images have CSS that modifies width OR height but not both; Next wants `width: auto` / `height: auto` to keep ratios.

## Design / IA on the public site

- **Footer Vision column is asymmetrical** — single link, while the Home/About columns each have three. Either expand or collapse.
- **Several routes are orphaned from the global nav**: `/ai`, `/open-source-health-design`, `/patient-engagement`, `/enterprise`, `/government`, `/why-hire-healthcare-design-studio` all render real pages but the header surfaces only Work / Services / About / Vision / Open Design / Contact. Decide which deserve nav slots, link the rest from `/services` cross-sells, or delete.
- **404 has no recovery CTA.** Just "NOT FOUND… the sadness." plus an illustration. Add a Home / Browse Work button — the global header is the only escape hatch right now.
- **Eric Topol testimonial pull-quote is enormous on mobile** — script font scales up rather than down; the quote spans 3+ screen heights at 390px. Knock the breakpoint sizing down a tier.
- **`/vision` hero label "Standardized Health Record"** wraps to "Standard- ized" mid-word. Set `hyphens: none` on those labels or shorten the string.
- **PersistentHero hydration gap** on `/work` — the doctor-with-glucose-monitor hero appears blank for ~1–2 seconds after navigation, then snaps in.

## Sanity Studio UX

- **Drag-to-order workflow is high-friction.** Banners on Case Study and Vision Piece say "5/31 pages still need to be ordered. Drag highlighted pages into place, then select Save New Order Preset." Every new doc requires the editor to (a) drag it into position and (b) explicitly persist the preset. The button isn't on the banner — editors have to remember where it lives. Five out of 31 case studies are currently in this stuck state, which is evidence the workflow doesn't survive contact with real editors. Recommend auto-saving on drop, or defaulting new docs to top/bottom and dropping the manual save step.
- **Document-list status dots are unlabeled.** Each row has a tiny dot (green / yellow / multi-dot) on the right with no tooltip and no legend. Editors will infer "green = published, yellow = draft" but it's not obvious — and "Untitled" rows have no dot at all. Add tooltips.
- **The top-right "Drafts" filter button is ambiguous.** It looks like it filters to drafts only when in fact it switches the whole structure pane's display mode. Rename to make the modes explicit (e.g. "Show: All / Published only / Drafts only").
- **Vision Piece "Featured" vs `/vision` "Spotlight" don't map 1:1.** The schema offers Featured / Non-featured; the public page also has a Spotlight section that appears to be sourced separately. Editors trying to "feature" a piece in Spotlight have no obvious path. Either unify under a single "Where shown" picker or document the relationship in the field description.
- **Team Member Bio is a full Portable Text field**, with Image and Columns buttons. For a 3-paragraph bio this is overkill and invites editors to drop in mismatched layouts. Restrict to `block` content with no headings, image, or columns — or use a plain `text`.
- **Health Visualization "Date" is a free-text string** ("Mar.2016", "Apr.2020", "Oct.2014" all coexist). Inconsistent formatting will leak through to the public site. Use a real Date type and format at render time.
- **Portable Text editor body window is short (~250px)** on large screens, with a small "fullscreen" icon at the end of the toolbar. For long case studies this is cramped; either grow the editor with content or default to a taller height.
- **Mobile Studio looks broken on first paint.** At 390px the structure pane is hidden, leaving an empty dark canvas with just the top bar. The hamburger surfaces tabs (Structure / Presentation / Getting Started / Feedback) and a theme picker, not the document list — there's no obvious hint that Structure is what gets you to the docs. Editors aren't going to author on phones, but the cold-start state could be improved with at least a placeholder.
- **Workspace name in the studio menu reads "MyStarrySpace"**, not GoInvo — looks like the Sanity Manage workspace name rather than the project name. Confusing for anyone seeing it for the first time.
